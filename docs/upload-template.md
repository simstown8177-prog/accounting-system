# 표준 업로드 양식

은행이나 카드사 원본 포맷을 직접 모두 지원하는 대신, 이 시스템은 먼저 `표준 업로드 양식`으로 복붙한 뒤 업로드하는 방식을 기준으로 합니다.

템플릿 파일:

- `templates/standard_ledger_upload.csv`

나중에 엑셀(`.xlsx`) 업로드를 붙이더라도 시트의 헤더는 이 템플릿과 동일해야 합니다.

## 필수 컬럼

| 컬럼명 | 설명 | 예시 |
|---|---|---|
| 거래일자 | 실제 승인일 또는 입출금일 | `2026-03-05` |
| 거래구분 | 회계 성격을 구분하는 핵심 값 | `신용카드사용` |
| 결제수단 | 현금, 계좌, 체크카드, 신용카드 등 | `신용카드` |
| 계좌/카드명 | 실제 자금수단 이름 | `OO카드` |
| 적요 | 거래 제목 | `식당 모임` |
| 수입금액 | 입금이면 양수, 아니면 `0` | `3200000` |
| 지출금액 | 지출이면 양수, 아니면 `0` | `90000` |

## 선택 컬럼

| 컬럼명 | 설명 |
|---|---|
| 상대처 | 거래처, 가맹점, 회사명 |
| 통화 | 기본 `KRW` |
| 카테고리 | 식비, 교통, 주거/공과금 등 |
| 세부카테고리 | 외식, 장보기, 버스 등 |
| 할부여부 | `Y`, `N` |
| 할부개월 | `3`, `6`, `12` 등 |
| 할부회차 | 현재 회차 |
| 원거래ID | 카드 사용, 결제, 상환 연결용 식별자 |
| 이자/수수료 | 할부이자, 연체이자, 수수료 |
| 잔액 | 계좌성 거래면 기입 권장 |
| 메모 | 자유 입력 |

## 거래구분 규칙

절대 섞으면 안 되는 구분들입니다.

| 거래구분 | 의미 | 처리 원칙 |
|---|---|---|
| 입금 | 월급, 환급, 수익 | 수입 |
| 일반계좌지출 | 현금성 즉시 지출 | 생활비 지출 |
| 계좌이체 | 계좌 간 이동 또는 송금 | 필요 시 자산이동 |
| 신용카드사용 | 카드 사용 시점 소비 | 생활비 지출 + 카드채무 발생 근거 |
| 카드결제 | 계좌에서 카드값 납부 | 새 소비가 아니라 부채상환 |
| 할부원금 | 할부 청구 중 원금분 | 부채상환 |
| 할부이자 | 할부에 붙는 금융비용 | 금융비용 |
| 수수료 | 은행/카드 수수료 | 금융비용 |
| 연체이자 | 연체 발생 비용 | 금융비용 |

## 입력 원칙

- `수입금액`과 `지출금액`은 동시에 값이 들어가면 안 됩니다.
- 신용카드 사용은 `신용카드사용`으로 넣고, 나중에 빠져나간 계좌 납부는 반드시 `카드결제`로 분리합니다.
- 할부 거래는 원거래와 월별 상환을 `원거래ID`로 연결합니다.
- 할부 이자와 수수료는 생활비 카테고리와 분리합니다.
- 은행 원본을 그대로 붙여 넣기 어렵다면 필요한 값만 이 양식으로 옮기면 됩니다.

## 권장 업로드 절차

1. 은행/카드사 거래내역을 내려받습니다.
2. `templates/standard_ledger_upload.csv` 헤더를 유지한 채로 행을 복붙합니다.
3. `거래구분`, `결제수단`, `계좌/카드명`을 정확히 채웁니다.
4. 카드 사용, 카드 결제, 할부 이자, 수수료를 별도 행으로 나눕니다.
5. 업로드 후 월 정산 보고서에서 미분류 거래만 검토합니다.
